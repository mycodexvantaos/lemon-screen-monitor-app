# ScreenMonitor App 設計文件

## 應用概述

**ScreenMonitor** 是一個 iOS 應用程式，用於即時監控另一方的手機螢幕操作。應用分為兩個角色：

1. **廣播方（被監控方）**：啟動螢幕共享，將自己的螢幕即時傳輸
2. **監控方（監控者）**：接收並觀看廣播方的螢幕，禁止任何遠端操作

---

## 螢幕清單

### 廣播方（被監控方）

| 螢幕名稱 | 功能 | 內容 |
|---------|------|------|
| **Disclaimer** | 首次啟動 | 詳細的免責聲明、隱私政策、法律告知 |
| **Broadcaster Home** | 主介面 | 啟動/停止廣播按鈕、連線狀態、房間 ID、QR 碼 |
| **Broadcasting Active** | 廣播中 | 廣播狀態指示、連線人數、停止廣播按鈕 |

### 監控方（監控者）

| 螢幕名稱 | 功能 | 內容 |
|---------|------|------|
| **Disclaimer** | 首次啟動 | 詳細的免責聲明、隱私政策、法律告知 |
| **Viewer Home** | 主介面 | 輸入房間 ID 或掃描 QR 碼、連線按鈕 |
| **Viewer Connecting** | 連線中 | 加載指示、連線狀態 |
| **Viewer Screen** | 監控中 | 全螢幕顯示廣播方的螢幕、斷開連線按鈕 |

## 主要使用者流程

### 流程 1：廣播方啟動螢幕共享

```
1. 廣播方首次打開應用
   ↓
2. 顯示免責聲明（需要確認同意）
   ↓
3. 進入 Broadcaster Home 螢幕
   ↓
4. 點擊「啟動廣播」按鈕
   ↓
5. iOS 系統彈出廣播選擇器
   ↓
6. 選擇此應用的 Broadcast Upload Extension
   ↓
7. 系統開始螢幕錄製（狀態列顯示紅色指示）
   ↓
8. 應用進入 Broadcasting Active 狀態
   ↓
9. 顯示房間 ID 與 QR 碼供監控方使用
```

### 流程 2：監控方連線並觀看

```
1. 監控方首次打開應用
   ↓
2. 顯示免責聲明（需要確認同意）
   ↓
3. 進入 Viewer Home 螢幕
   ↓
4. 選擇連線方式：
   a) 手動輸入房間 ID
   b) 掃描廣播方的 QR 碼
   ↓
5. 點擊「連線」按鈕
   ↓
6. 進入 Viewer Connecting 狀態
   ↓
7. 連線成功後進入 Viewer Screen
   ↓
8. 全螢幕顯示廣播方的即時螢幕
   ↓
9. 監控方無法進行任何操作（禁止點擊、滑動等）
```

### 流程 3：斷開連線

```
廣播方：點擊「停止廣播」或通過控制中心停止
  ↓
伺服器通知監控方
  ↓
監控方返回 Viewer Home
```

## 主要內容與功能

### 廣播方功能

| 功能 | 描述 |
|------|------|
| **啟動廣播** | 觸發 iOS 系統廣播選擇器，開始螢幕共享 |
| **停止廣播** | 停止螢幕共享 |
| **房間 ID** | 唯一識別符，監控方用於連線 |
| **QR 碼** | 編碼房間 ID，方便監控方快速掃描連線 |
| **連線狀態** | 顯示當前有多少監控方已連線 |
| **免責聲明** | 首次啟動時強制顯示，說明隱私風險與法律責任 |

### 監控方功能

| 功能 | 描述 |
|------|------|
| **房間 ID 輸入** | 手動輸入房間 ID 進行連線 |
| **QR 碼掃描** | 使用相機掃描廣播方的 QR 碼 |
| **即時螢幕顯示** | 全螢幕顯示廣播方的螢幕，解析度 720p |
| **禁止操作** | 所有觸控事件被禁用，無法進行任何操作 |
| **斷開連線** | 停止接收螢幕流並返回主介面 |
| **免責聲明** | 首次啟動時強制顯示，說明隱私風險與法律責任 |

## 免責聲明內容方向

應用首次啟動時顯示的免責聲明應包含：

1. **隱私警告**：此應用將捕獲並傳輸使用者的整個螢幕，包括所有個人資訊
2. **法律責任**：使用者對自己的行為負全責，未經他人同意進行監控可能違反法律
3. **知情同意**：被監控方必須知道並同意進行螢幕共享
4. **資料安全**：螢幕流通過網路傳輸，可能存在安全風險
5. **禁止用途**：禁止用於非法監控、詐騙、騷擾等不當目的

## 技術實作考量

### 廣播端（被監控方）

- 使用 Broadcast Upload Extension 進行系統級螢幕捕獲
- 通過 WebSocket 連接至伺服器
- 發送 H.264 編碼的視訊流（720p, 20-30 fps）
- 房間 ID 由伺服器分配，QR 碼在應用內生成

### 監控端（監控者）

- 使用 WebRTC 接收視訊流
- 全螢幕顯示，禁用所有觸控事件
- 顯示連線狀態與錯誤提示

### 伺服器

- 管理房間與連線
- 轉發視訊流（SFU 模式）
- 處理連線與斷開事件

## 設計原則

1. **簡潔性**：介面清晰，操作流程直觀
2. **安全性**：強調隱私風險與法律責任
3. **可用性**：大按鈕、清晰的狀態指示
4. **一手操作**：所有控制元素位於螢幕下半部分，便於單手使用
5. **iOS 風格**：遵循 Apple Human Interface Guidelines，確保應用看起來像原生 iOS 應用
